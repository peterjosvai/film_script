I mean, that it deserves some thinking and development...

With a typewriter it's all simple, and just typing "TEXT OVER BLACK" in the action "lane" could have been suffice,

Plus, we are creating a notation system... so it's absolutely worth thinking  about possible "solutions"...

## on-screen text types

**in-scene** and **scene-level** ON-SCREEN TEXT

## The other thing: perspectives!

Film_script isn't meant for wannabe screenwriters aspiring to settle in Hollywood or Bollywood .:/ 
but for artists with visions

Therefore, the goal is NOT only producing a decent looking script... 
But also to move beyond that...

For example: changing font-size for scene-heading

Although it's technically zero problem using css (style sheets), 
it takes a lot of thinking... and consideration... 

Cause the Hollywood screenplay format is not Hollywood's.. 
it is universal, like English language... 
This means that it's worth to take all rules seriously... 
before we "touch" them :)

It took me weeks to settle with 110% font sizes for scene heading...
It is not a rule or a new rule...
but a new alternative... (like using light gray background with scene headings, which CELTX does)
and it cannot bu just an idea... 
it has to harmonize with the existing notation system well :) 

## ALL in all... 

As I am developing the converter, I'm also developing the notation concept... 
which is simple and complex at the same time... 

simplicity of the film_script notation system (formatting rule-set) involves
being simple to use... 

I mean, while with Final Draft (TM)(R) one can write effortlessly, it cannot be that
with a simple text editor, using the film_script formatting, we have to do
complicated formatting actions :) :) 

Our system has to be simpler, faster, and more intelligent... 
providing a more fluent working style...

INTELLIGENT: allowing the writer to write whatever s/he wants... in whatever way or style...
without feeling constrained in a  framework... as if filling forms....

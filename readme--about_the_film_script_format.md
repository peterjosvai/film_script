### **Fountain** -- the format / working method

Around 2010 / 2012 (I'm not sure) **Fountain** was born. (see: [https://fountain.io/](https://fountain.io/). The creators of **Fountain** thought up a formatting system, a simple set of document formatting rules, a variant of markdown, which can be used for writing screenplays with ease.

### film_script -- the format / working method

Just like Fountain, **film_script**, too, is a simple set of formatting rules. Film_Script was and is inspired by the Fountain concept. What made film_script necessary (in 20126) is that Fountain needs dedicated Fountain software, which is rare.

It can also be used with any text editor, but I found it a cumbersome task. Writing a screenplay demands distraction-free typing.

The **film_script** format (set of formatting rules) makes it possible to write fluently, without distraction.

Film_script is not to replace Fountain at all. Fountain has been a revolution. The **film_script** workflow is based on the fact that the Fountain format is accepted in Celtx, and can be exported to Final Draft format.

The film_script format has two variants, one for simple text editors (like Mousepad or Xed, or Gedit or Kate or  Geany or Sublime, etc. But GNU nano also works!), and one which can be used to write screenplays in **Libreoffice Writer**.

The variant that works with a text editor, is **indentation-based**, inspired by Python :) This variant is referred to as the **film_script_i** format.

The other, which can be used in Libreoffice Writer, is heading-based. It is referred to as the **film_script_h** format.

I writer, we can save it as **.odt** or as **.md** (markdown). As long as we work on it, it can be the **.odt** file, exporting means saving it as a markdown document with **.md** extension. (possible in Writer 26.xx and above).
